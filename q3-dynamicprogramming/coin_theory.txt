Question 3 – Coin Change (Minimum Coins)

1. Recurrence Relation:
Let dp[i] be the minimum number of coins required to make amount i.

dp[0] = 0
dp[i] = min(dp[i - coin] + 1) for all coins where coin <= i

2. Bottom-Up Approach:
We build the solution from 0 to S using the recurrence relation.

3. DP Table for {1,3,4,5}, S = 7:
i:   0 1 2 3 4 5 6 7
dp:  0 1 2 1 1 1 2 2

4. Reconstruction:
Store the last coin used in a parent array.
Backtrack from S to 0 to retrieve the coins.

5. Time Complexity:
O(n * S), where n = number of coins

Space Complexity:
O(S)

6. Why Greedy Fails:
Greedy picks the largest coin first.

Example:
Coins = {1, 3, 4}, S = 6

Greedy:
4 + 1 + 1 = 3 coins

Optimal:
3 + 3 = 2 coins

Thus greedy does not always give optimal solution for arbitrary denominations.
