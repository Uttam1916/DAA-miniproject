Question 2 – Greedy Method
Prim's Minimum Spanning Tree

----------------------------------------------------------------

1. Greedy Choice Property:

At each step, select the minimum weight edge that connects a vertex in the current tree to a vertex outside it.

Proof:
Let the graph be divided into two sets:
- S (vertices included in MST)
- V − S (remaining vertices)

Consider the minimum weight edge (u, v) crossing this cut.

If this edge is not in the MST:
- Adding it creates a cycle
- In that cycle, there must exist an edge with weight greater than or equal to (u, v)
- Removing that edge keeps the graph connected and does not increase total weight

Therefore, choosing the minimum crossing edge is always safe and leads to an optimal MST.

----------------------------------------------------------------

2. Algorithm Design:

Prim’s Algorithm:
- Start from any vertex (in code: vertex 0, i.e., A)
- Maintain an array of keys (minimum edge weights)
- Maintain a visited set (mstSet[])
- Repeatedly pick the vertex with the smallest key value
- Update adjacent vertices
- Continue until all vertices are included

Kruskal’s Algorithm:
- Sort all edges by weight
- Pick the smallest edge
- Add it if it does not form a cycle (using Union-Find)
- Repeat until MST is formed

O(E log E)
